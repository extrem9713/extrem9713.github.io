這個是我在老爸私房錢上傳heroku時所遇到問題，後來將error: src refspec master does not match any 這段話上網google，得到的答案是:

但是根據其操作還是無法上傳

況且我也確定資料夾內已有資料，而且都有進行commit，不經認為是我搜尋的答案有問題

後來看到slack有人跟我有相同的問題時，才找到正確的解法

原來只是目錄名稱的不同，只要把 git push heroku master 改為 git push heroku main就能正確執行

感想

即使只是個小問題，也讓我卡關半天，自行google也找錯了方向，運氣好看到slack上也有人跟我也相通問題才解決，會犯這個錯誤的原因，我歸咎於自己還是對後端運作機制沒有相當熟悉，只看著教案進行實作，才會產生這個問題。問題本身就是目錄的名稱不同所致，但是自己都沒有察覺到，表示自記對於終端機上的內容其實也不適相當清楚，看來自己還需要再多加努力，才能更加了解後多。
